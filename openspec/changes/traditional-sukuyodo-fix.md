# 宿曜道傳統規則修正

## 變更內容

將系統的三處計算邏輯修正為符合宿曜道傳統規則：

### BUG 1: 不成就日改用農曆月日（嚴重度: 中）

`is_fujoubyou()` 目前使用西曆月日判定，但不成就日傳統上基於農曆月日。
以 1977/10/29 為例，西曆判定「非不成就日」，農曆判定「是不成就日」，結果完全相反。

### BUG 2: 五行相性加入方向性（嚴重度: 低-中）

`_calc_fortune_element_relation()` 對 generating/weakening 使用雙向比對，
但兩者的元素對完全重疊（只是方向相反），導致 weakening(-5) 永遠不觸發。

修正為方向性判定：
- `elem2 → elem1`（日元素生本命）= generating(+10)
- `elem1 → elem2`（本命生日元素）= weakening(-5)

### 改善: 一粒萬倍日改用節月（嚴重度: 低）

一粒萬倍日傳統上以二十四節氣的「節月」為月份基準，非西曆月份。
月初約 4-7 天的判定會不同。

## 影響範圍

- `backend/services/japanese_calendar.py` — 不成就日、一粒萬倍日
- `backend/services/sukuyodo.py` — 五行相性方向性

## 測試計畫

1. 不成就日: 用已知農曆日期交叉驗證（如 1977/10/29 農曆九月十七應為不成就日）
2. 五行相性: 確認 20 對五行組合中 weakening 正確觸發（5 對）
3. 一粒萬倍日: 對比節月與西曆月在月初的判定差異
4. 回歸測試: 確認每日/週/月/年運勢計算無異常

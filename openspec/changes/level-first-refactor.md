# 運勢計算回歸原典：等級優先制重構

## 問題分析

目前的運勢計算以「分數」為核心，所有邏輯（凌犯逆轉、甘露/羅刹、六害宿）都在數值層面操作。但宿曜經原典從來沒有分數的概念，只有吉凶等級的質性判斷。

現行問題：
1. 凌犯逆轉力道（-20/-12/-5/+15）無經典依據，是自行定義
2. 多因素數值疊加產生非直覺結果（如 6/26 甘露逆轉+安壊緩和+六害宿互相抵消）
3. 分數給予虛假精確度（85 vs 83 在原典中無差異）

## 變更內容

將計算核心從「分數驅動」改為「等級驅動」：

1. **等級判定**（原典邏輯）：本命宿 x 當日宿 → 關係等級
2. **等級翻轉**（凌犯/甘露/羅刹在等級層面操作）
3. **等級→分數映射**（最後一步，僅供 UI 顯示用）

### 等級體系

| 等級 | 對應關係 | 顯示分數 |
|------|---------|---------|
| 大吉 | 栄親 | 90 |
| 吉 | 業胎/命 | 75 |
| 中吉 | 友衰 | 60 |
| 小凶 | 危成 | 45 |
| 凶 | 安壊 | 35 |

### 凌犯翻轉規則（原典）

| 原等級 | 凌犯中 |
|--------|--------|
| 大吉 | → 凶 |
| 吉 | → 小凶 |
| 中吉 | → 中吉（平不變） |
| 小凶 | → 吉 |
| 凶 | → 大吉 |

### 特殊日（凌犯中）

- 甘露日：大吉 → 凶
- 羅刹日：凶 → 吉
- 金剛峯日：不受影響

### 移除的項目

- `RELATION_SCORE_RANGES`（分數範圍，改用等級映射）
- 元素微調的分數加減（改為等級內的修飾詞）
- 凌犯的分段減分邏輯（-20/-12/-5/+15）
- 六害宿的 severity 加權減分

### 保留的項目

- 七曜元素相性（作為輔助描述，不影響等級）
- 三期サイクル（獨立系統，不受影響）
- 九曜流年（獨立系統，保留既有分數邏輯）
- 分類運勢（career/love/health/wealth）改為從等級衍生

## 影響範圍

- `backend/services/sukuyodo.py` — 核心重構（calculate_daily_fortune 等）
- `backend/data/sukuyodo_fortune.json` — 可能需調整元素加成數值
- `frontend/src/composables/useSukuyodo.ts` — 適配新回傳格式
- `frontend/src/components/` — 日/週/月/年運勢顯示元件

# 吉日判斷衝突解決與師徒關係類型

## 問題背景

### 雙人吉日 `get_pair_lucky_days` — 完全沒有過濾

判斷吉日時只看「雙方與當日宿的關係類型」和「雙方日運平均分數」，以下因素全部缺失：

| 缺失項目 | 原典依據 | 風險 |
|---|---|---|
| 凌犯期間 | T21 p.391b「犯業宿及榮安成友親，所求不遂」| 吉凶逆轉的日子被推薦 |
| 六害宿 | T21 p.392b「七曜犯命宿則亡失錢財必多災厄」| 災厄日被推薦 |
| 羅刹日 | T21 p.392c「不宜舉動百事，必有殃禍」| 凶日被推薦 |
| 暗黒の一週間 | 三期 distance 9-15，壊方向低潮 | 低迷期被推薦 |
| 壊の日 | T21 p.391b「壞宿日，不宜遠行出入遷移」| 凶日被推薦 |
| 危の日 | T21 p.391b「危壞日，不宜遠行、買賣、裁衣」| 不穩定日被推薦 |
| 金剛峯日 | T21 p.398b「宜作一切降伏法，持日天子呪」| 吉日加分未計入 |
| 甘露日 | T21 p.398b「宜受灌頂法、受戒、出家修道」| 吉日加分未計入 |
| 日宿吉凶 | 各宿各有宜忌事項 | 行動適配未利用 |
| 等級過低 | 小凶/凶的日子 | 低分日可能被推薦 |

### 個人吉日 `get_lucky_days` — 做了一半

| 項目 | 狀態 | 說明 |
|---|---|---|
| 凌犯排除 | 有（旗標制） | 依賴 `avoid_ryouhan: True`，需每個 action 手動設定 |
| 暗黒排除 | 有（旗標制） | 依賴 `avoid_dark_week: True` |
| 羅刹排除 | 有（旗標制） | 依賴 `avoid_rasetsu: True` |
| 等級過低 | 有 | 小凶/凶直接排除 |
| 六害宿 | **沒有** | 個人也缺 |
| 壊/危の日 | **沒有** | 靠分數間接過濾，不精確 |
| 金剛峯/甘露加分 | **沒有** | |
| 日宿吉凶屬性 | **沒有** | |
| 衝突標記 | **沒有** | 正反因素同時存在時無法判斷 |

### 關係類型缺少「師徒」

無法用於傳法、灌頂、授戒等宗教儀式的吉日判斷。

## 變更內容

### 1. 後端：統一吉日過濾邏輯 (`sukuyodo.py`)

新增私有方法 `_evaluate_day_quality`，個人和雙人吉日共用：

```python
def _evaluate_day_quality(self, daily_fortune: dict) -> dict:
    """
    評估單日品質，回傳排除/降級/加分/衝突資訊。

    Returns:
        {
            "excluded": bool,           # 是否硬排除
            "exclude_reason": str,      # 排除原因
            "downgrade": bool,          # 是否降級
            "downgrade_reasons": [],    # 降級原因列表
            "upgrade": bool,            # 是否升級
            "upgrade_reasons": [],      # 升級原因列表
            "conflicts": {              # 衝突標記（正反同時存在）
                "positive": [],
                "negative": [],
                "verdict": str          # "建議避開" / "謹慎行事" / "可行"
            } | None
        }
    """
```

#### 硬排除（原典依據）

| 條件 | 原典 | 說明 |
|---|---|---|
| 凌犯期間 | T21 p.391b「犯業宿及榮安成友親，所求不遂」| 吉凶逆轉，不可預期 |
| 六害宿日 | T21 p.392b「七曜犯六宮宿者必有災厄」| 命/意/事/克/聚/同六宮 |
| 羅刹日 | T21 p.392c「不宜舉動百事，必有殃禍」| 全面凶日 |
| 壊の日 | T21 p.391b「壞宿日，不宜遠行出入遷移、買賣裁衣、剃頭剪甲」| 經文明確列為不宜 |

注意：凌犯期間的羅刹日逆轉為吉，但凌犯本身已排除，所以不會衝突。

#### 降級（一級：大吉→吉，吉→中吉）

| 條件 | 原典 | 說明 |
|---|---|---|
| 暗黒の一週間 | 三期 distance 9-15 | 破壊の週中最低迷的 7 天 |
| 危の日 | T21 p.391b「危壞日，不宜遠行、買賣」| 與壊同列，但程度較輕 |
| 衰の日 | T21 p.391b「衰危壞宿日，並不宜」| 經文明確三者並列 |

原典 1-3 節：「衰、危、壞宿日，並不宜遠行，出入遷移、買賣裁衣、剃頭剪甲並不吉。」
壊最重（硬排除），衰/危次之（降級）。

#### 加分（一級：中吉→吉，吉→大吉）

| 條件 | 原典 | 適用行動 |
|---|---|---|
| 甘露日 | T21 p.398b「宜受灌頂法、受戒、出家修道，一切並吉」| 傳法/灌頂/授戒/修行/學習 |
| 金剛峯日 | T21 p.398b「宜作一切降伏法，持日天子呪」| 降伏/護摩/密教修法 |
| 業宿日 | T21 p.391b「業宿直日，所作皆吉祥」| 所有行動 |
| 栄の日 | 三期最佳日 | 所有行動 |
| 親の日 | 三期吉日 | 人際/結交相關 |
| 友の日 | T21 p.391c「友親日，宜結交朋友大吉」| 人際/結交相關 |
| 日宿利佛事 | 各宿吉凶表 | 佛事/祭祀類行動 |

加分前提：無硬排除條件、無降級條件（否則變成衝突）。

#### 衝突判定

當加分因素和降級/排除因素同時存在時：

```python
# 範例：3/9 金剛峯日 + 暗黒 + 危の日
conflicts = {
    "positive": ["金剛峯日（宜降伏法、持呪）", "心宿日（宜佛事）"],
    "negative": ["暗黒の一週間", "危の日（不宜遠行買賣）"],
    "verdict": "建議避開"  # 負面因素 >= 正面因素
}
```

verdict 邏輯：
- 正面 > 負面 → 「謹慎可行」
- 正面 = 負面 → 「建議避開」
- 正面 < 負面 → 「建議避開」
- 有硬排除 → 永遠「建議避開」

### 2. 後端：重構個人吉日 (`sukuyodo.py`)

`get_lucky_days` 中移除 `avoid_ryouhan`/`avoid_dark_week`/`avoid_rasetsu` 旗標制，改為呼叫統一的 `_evaluate_day_quality`。所有 action 自動套用完整過濾，不再需要手動設旗標。

新增個人吉日也缺少的：
- 六害宿排除
- 壊/危/衰の日處理
- 金剛峯/甘露加分
- 衝突標記

### 3. 後端：新增 `master` 關係類型 (`sukuyodo.py`)

```python
"master": {  # 師徒
    "name": "師徒",
    "actions": [
        {
            "key": "dharma_transmission",
            "name": "傳法",
            "favor_relations": ["eishin", "mei", "gyotai"],
            "favor_score": 85,
            # 原典：甘露日「宜受灌頂法、受戒、習學經法」
            # 傳法為授受經法之核心，與甘露日吉義相合
        },
        {
            "key": "initiation",
            "name": "灌頂",
            "favor_relations": ["eishin", "mei"],
            "favor_score": 85,
            # 原典：甘露日「宜冊立、受灌頂法」— 明確記載
            # 金剛峯日「宜作一切降伏法」— 密教灌頂含降伏意
        },
        {
            "key": "ordination",
            "name": "授戒",
            "favor_relations": ["eishin", "mei", "gyotai"],
            "favor_score": 80,
            # 原典：甘露日「受戒、出家修道，一切並吉」— 明確記載
        },
        {
            "key": "teaching",
            "name": "教學",
            "favor_relations": ["eishin", "gyotai", "yusui"],
            "favor_score": 70,
            # 原典：甘露日「習學經法」
            # 成日(T21 p.391b)「修學問道、合藥求仙吉」
        },
    ]
}
```

師徒行動額外規則：
- 傳法/灌頂/授戒：**雙方分數都必須 >= 60**（不只看平均，傳法是雙向輸出）
- 甘露日對傳法/灌頂/授戒：額外加分（原典明確記載）
- 金剛峯日對灌頂/降伏類：額外加分（原典「宜作一切降伏法」）
- 日宿利佛事/祭祀：額外加分
- 成の日對教學：額外加分（原典「成日，修學問道吉」）

# 三期/特殊日系統原典校對修正

## 背景

使用者（高野山真言宗傳法灌頂阿闍梨）指出：系統將「暗黒の一週間」整週標為禁忌，但原典 T21n1299 逐日記載各有不同吉凶，安日「作壇場並吉」、成日「修道學問、作諸成就法並吉」、壊日「宜作鎮壓、降伏」。系統過度簡化導致判斷失準。

## 問題清單

### P0 — 原典不符（必修）

| # | 問題 | 原典依據 | 現狀 | 影響 |
|---|------|---------|------|------|
| 1 | 暗黒の一週間整體排除 | 二九各日吉凶不同（p.397c-398a） | distance 9-15 全部 excluded=True | 安日/成日被錯誤排除，吉日遺漏 |
| 2 | 壊の日整體排除 | 「宜作鎮壓、降伏怨讎」(p.398a) | day_type=="壊の日" → excluded=True | 降伏護摩日被排除 |
| 3 | 金剛峯日缺護摩 | 「及作護摩，並諸猛利等事」(p.398b-c) | 描述只提「修法、寫經、授戒」 | 阿闍梨查護摩日找不到金剛峯日 |

### P1 — 描述偏離（應修）

| # | 問題 | 原典依據 | 現狀 |
|---|------|---------|------|
| 4 | 命日/胎日禁忌弱化 | 「不宜舉動百事」(p.391b) | 描述為「重新審視原點」「播種」，未強調禁忌 |
| 5 | 業日嚴重性弱化 | 「所作善惡亦不成就，甚衰」(p.397c) | 描述為「面對拖延課題」 |
| 6 | 衰日缺「解除諸惡」 | 「唯宜解除諸惡、療病」(p.391b) | 描述為「內省和學習」，缺實踐層面 |
| 7 | 安日缺「作壇場吉」 | 「作壇場並吉」(p.397c) | 描述為「穩定安寧，適合日常事務」 |
| 8 | 成日缺「修道」強調 | 「修道學問、作諸成就法並吉」(p.398a) | 描述為「事情容易成就，適合收尾」 |
| 9 | 凌犯化解未突出護摩 | 「福者，謂入灌頂及護摩」(p.392a-b) | 凌犯建議偏重「吉凶逆轉」，未明示護摩為化解法 |

### P2 — 前端顯示調整

| # | 問題 | 現狀 |
|---|------|------|
| 10 | 暗黒の一週間視覺過度負面 | 紅色/紫色標記暗示全面凶，但安日/成日有吉相 |
| 11 | ICS 暗黒建議文字過於籠統 | 「低潮期，別急著推進新東西」不分日型 |

## 修正方案

### 後端 sukuyodo.py

#### 1. 廢除「暗黒の一週間整體排除」邏輯

**`_evaluate_day_quality()` (~行 4662-4666)**

現狀：
```python
if sanki.get("is_dark_week", False):
    if not excluded:
        excluded = True
        exclude_reason = "暗黒の一週間，能量低迷期，建議避開"
    conflicts.append("暗黒週")
```

改為：依三九日型逐日判斷
```python
# 暗黒の一週間不再整體排除，改為逐日判斷
# 業日：excluded（所作不成就）
# 衰日：rating_shift -= 1（療病解惡可，餘事不宜）
# 危日：rating_shift -= 1（婚宴可，餘事謹慎）
# 安日：不排除（作壇場吉）
# 栄日：不排除（諸吉事大吉）
# 成日：不排除（修道成就法吉）
# 壊日：不全排除，改為 conditional（降伏可，餘不堪）
if sanki.get("is_dark_week", False):
    conflicts.append("破壊の週")
    # 不再 excluded=True，交由日型判斷
```

#### 2. 修改壊の日排除邏輯

**`_evaluate_day_quality()` (~行 4647-4651)**

現狀：
```python
if day_type == "壊の日":
    excluded = True
    exclude_reason = "壊の日，萬事不宜"
```

改為：
```python
if day_type == "壊の日":
    # 原典：「宜作鎮壓、降伏怨讎及討伐，餘並不堪」
    # 降伏法/降伏護摩可，但一般吉日排除
    if not excluded:
        excluded = True
        exclude_reason = "壊の日，降伏法可行，餘事不宜"
    conflicts.append("壊日")
```

注意：壊日仍排除一般吉日，但在護摩/降伏相關分類中應允許。需新增判斷 action 是否為降伏類。

#### 3. 吉日篩選邏輯調整

**`get_lucky_days()` 的 clean_high 過濾 (~行 4033-4038)**

現狀：
```python
clean_high = [d for d in all_daily
              if d["score"] >= 70
              and not d.get("is_dark_week", False)  # ← 整週排除
              ...]
```

改為：移除 is_dark_week 整體排除，由 _evaluate_day_quality 的逐日判斷處理。

#### 4. 更新 SANKI_DAY_TYPES 描述文字

逐項對照原典修正（行 1622-1662）：

| 日型 | 修正重點 |
|------|---------|
| 命の日 | 加入「不宜舉動百事」禁忌 |
| 業の日 | 加入「所作善惡亦不成就，甚衰」 |
| 胎の日 | 加入「不宜舉動百事」禁忌 |

| 成の日 | 補充「修道學問、合和長年藥法、作諸成就法」 |
| 壊の日 | 補充「宜作鎮壓、降伏怨讎」，明確區分降伏可/餘不堪 |

# 查詢本命宿對話框重新設計

## 現況問題

1. **核心 UX 問題**: 對話框把「查詢任何人」跟「我的檔案管理」混在一起
2. 想幫不相關的人查本命宿，沒有明確的入口
3. 查詢非收藏對象的日期會覆蓋用戶自己的生日（Bug）
4. 新增/編輯表單內嵌展開後，對話框過長
5. 快速選擇 + 日期輸入 + 收藏列表 + 新增表單全部擠在一起

## 變更內容

### 核心改動

1. **對話框只做一件事：輸入日期查詢本命宿**
   - 任何人都能查，不限於已儲存的人
   - 查詢結果不自動覆蓋用戶自己的生日

2. **快速選擇只是捷徑，不是主角**
   - 放在日期輸入下方，以 chip 形式呈現
   - 沒有收藏對象時不顯示，介面更乾淨

3. **收藏對象管理移到折疊區**
   - 預設折疊，不干擾主要查詢流程
   - 新增/編輯用獨立子對話框

4. **修正生日覆蓋 Bug**
   - 查詢時不再自動儲存為「我的生日」
   - 如果用戶尚未設定自己的生日，查詢後提示「要儲存為我的生日嗎？」

```
│                                  │
│  西曆生日                        │

│  [我] [正念] [樂賢]              │  ← 快速選擇 chip
│                                  │

└──────────────────────────────────┘
```

展開收藏：

```
│  ▾ 收藏對象管理 (2)              │
│  ┌──────────────────────────┐    │
│  │ 正念  1991-01-27  交往   ✎🗑 │
│  │ 樂賢  1996-08-21  配偶   ✎🗑 │
│  └──────────────────────────┘    │
│              [+ 新增對象]        │
```

子對話框（新增/編輯）：

```
┌────────────────────────────┐
│  新增收藏對象             X │
├────────────────────────────┤
│  暱稱    [            ]    │
│  生日    [     📅     ]    │
│  關係    [朋友/同事   ▾]    │
├────────────────────────────┤
│             [取消] [儲存]   │
└────────────────────────────┘
```

### 改動重點

| 項目 | 現況 | 改為 |
|------|------|------|
| 日期輸入位置 | 快速選擇下方 | 最上方（主角） |
| 快速選擇 | 上方，label「快速選擇：」 | 日期下方，無 label，chip 風格 |
| 說明文字 | 長段文字 | 移除（介面自明） |
| 收藏對象 | 平鋪展開 | sl-details 折疊 |
| 新增/編輯表單 | 內嵌展開 | 獨立子對話框 |
| 查詢後儲存 | 自動覆蓋生日 | 不自動儲存 |

## 影響範圍

- `frontend/src/views/HomeView.vue` — 模板重構 + 樣式
- `frontend/src/composables/useSukuyodo.ts` — 移除 lookupMansion 中的自動儲存邏輯

## 設計規格

- 間距: 4px 倍數 (8/12/16/24px)
- 折疊元件: `<sl-details>` (Shoelace 原生)
- 子對話框: `<sl-dialog>` (小尺寸)
- 色彩: 沿用 CSS 變數 (--accent, --bg-elevated, --border)
- chip 按鈕: 圓角 pill 形，未選中灰底、選中 accent

## 測試計畫

1. 輸入任意日期可查詢，不會覆蓋自己的生日
2. 快速選擇 chip 點擊後自動帶入日期（不直接查詢）
3. 收藏對象折疊/展開正常
4. 新增/編輯/刪除收藏對象在子對話框中運作正常
5. 手機版對話框高度合理
